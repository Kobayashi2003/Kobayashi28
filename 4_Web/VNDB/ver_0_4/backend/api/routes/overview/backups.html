Backups API Documentation

This document describes the endpoints available for managing backups in the system.

GET /backups

Retrieve a list of all backups.
Response: Returns a task ID for asynchronous processing.

GET /backups/{id}

Retrieve information about a specific backup.
Response: Returns a task ID for asynchronous processing.

POST /backups

Create a new backup of the current system state.
Response: Returns a task ID for asynchronous processing.

POST /backups/{id}

Restore the system to the state saved in the specified backup.
Response: Returns a task ID for asynchronous processing.

DELETE /backups

Delete all backups in the system.
Response: Returns a task ID for asynchronous processing.

DELETE /backups/{id}

Delete a specific backup.
Response: Returns a task ID for asynchronous processing.

Notes

  • All endpoints return a task ID immediately and perform the requested operation asynchronously.
  • The status and result of each operation can be checked using the Tasks API.
  • The {id} in the paths represents the unique identifier of a specific backup.
