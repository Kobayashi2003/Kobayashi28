VNDB Database Routes Summary

Backup and Restore

POST /database/backup

Description: Create a database backup

Parameters: Optional 'filename' in JSON body

POST /database/restore

Description: Restore the database from a backup

Parameters: 'filename' in JSON body

Cleanup

POST /database/cleanup/<cleanup_type>

Description: Perform database cleanup

Valid cleanup types: vn, tag, producer, staff, character, trait, all

CRUD Operations

GET, POST, PUT, DELETE /database/crud/<model_type>/<id>

Description: Perform CRUD operations on database entries

Valid model types: vn, tag, producer, staff, character, trait

Parameters: Varies based on the HTTP method

Create

POST /database/create/<create_type>

Description: Create a new database entry

Valid create types: vn, tag, producer, staff, character, trait

Read

GET /database/read/<read_type>

GET /database/read/<read_type>/<id>

Description: Read database entries

Valid read types: vn, tag, producer, staff, character, trait

Update

PUT /database/update/<update_type>/<id>

Description: Update a database entry

Valid update types: vn, tag, producer, staff, character, trait

Delete

DELETE /database/delete/<delete_type>/<id>

Description: Delete a database entry

Valid delete types: vn, tag, producer, staff, character, trait

Error Handling

The database blueprint includes error handlers for 400 (Bad Request), 404 (Not Found), and 500 (Server Error) status codes.

Notes

  • All routes return a task ID for asynchronous processing.
  • The CRUD route combines functionality for Create, Read, Update, and Delete operations.
  • Type validation is performed for all routes that accept type parameters.
  • ID validation is performed using the infer_type_from_id function.
