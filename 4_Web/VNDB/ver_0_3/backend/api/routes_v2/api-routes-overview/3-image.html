VNDB Image Routes Summary

Download Images

POST /image/download/<download_type>/<id>

Description: Download images for a VN or character

Valid download types: vn, character

ID format: 'v' for VN, 'c' for character

Upload Images

POST /image/upload/<upload_type>/<id>

Description: Upload images for a VN or character

Valid upload types: vn, character

ID format: 'v' for VN, 'c' for character

Parameters: 'files[]' in request.files (multiple file upload)

Update Images

POST /image/update/<update_type>/<id>

Description: Update images for a VN or character

Valid update types: vn, character

ID format: 'v' for VN, 'c' for character

Delete Image

DELETE /image/delete/<delete_type>/<id>

Description: Delete a single image

Valid delete types: vn, character

ID format: Image ID

Delete All Images

DELETE /image/delete/batch/<delete_type>/<id>

Description: Delete all images for a VN or character

Valid delete types: vn, character

ID format: 'v' for VN, 'c' for character

Get Image

GET /image/get/<get_type>/<id>

Description: Get a single image

Valid get types: vn, character

ID format: Image ID

Note: This route is cached for 60 seconds

Get All Images

GET /image/get/batch/<get_type>/<id>

Description: Get all images for a VN or character

Valid get types: vn, character

ID format: 'v' for VN, 'c' for character

Note: This route is cached for 60 seconds

Serve Image

GET /image/v/<id>

GET /image/c/<id>

Description: Serve a specific image file

ID format: Image ID

Note: 'v' for VN images, 'c' for character images

Error Handling

The image blueprint includes error handlers for 400 (Bad Request), 404 (Not Found), and 500 (Server Error) status codes.

Notes

  • All routes (except serve_image) return a task ID for asynchronous processing.
  • ID validation is performed using is_valid_id and infer_type_from_id functions.
  • Image ID validation is performed using is_valid_image_id function.
  • The serve_image route uses caching for image ID and path retrieval.
